Why I Fell in Love with Clean Code

Published on December 4, 2025 | Category: Tech

When I first started coding, my scripts were messy. Variables had names like x1, temp, and functions did too many things at once. It worked, but it wasn’t readable — not for me, and certainly not for anyone else.

Over time, I realized that writing clean code isn’t just about making things look pretty. It’s about respect: respect for anyone who reads your code, respect for your future self, and respect for the craft of programming itself. Clean code allows me to think more clearly, debug faster, and build features with confidence.

Some lessons I learned along the way:

  • Meaningful names: Use clear variable and function names — it’s like documenting your code without comments.
  • Single responsibility: Each function should do one thing and do it well.
  • Readable formatting: Consistent indentation, spacing, and structure help your code breathe.
  • Refactoring: Don’t be afraid to revisit old code and make it cleaner.

By focusing on these principles, I started to enjoy coding more. Mistakes became easier to spot, collaboration with classmates improved, and projects felt… alive. Writing clean code isn’t just a habit; it’s a mindset that shapes how I approach every problem.

And that’s why I fell in love with clean code — not for the perfection it brings, but for the clarity, care, and creativity it allows.

My Group Project

Published on December 4, 2025 | Category: Tech

In our collaborative project, we explored modern web development using HTML, CSS, and JavaScript. Each team member contributed unique skills — from design to coding — resulting in a functional, interactive website.

Key lessons learned from the project:

  • Teamwork: Communication and coordination are crucial to success.
  • Version control: Using Git ensured we didn’t overwrite each other’s work.
  • Problem-solving: Collaborating on challenges taught us creative solutions.
  • Presentation: Documenting our work and sharing demos improved clarity.
